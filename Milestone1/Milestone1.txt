D4 - In a paragraph and/or a handful of bullet points, compare and contrast user and kernel
threads providing an example problem for each where it is favored over the other. [4 pts]

User threads are managed by user space libraries, entirely in user space. The operating system only views the process as a single-threaded process and can't see the other threads. With user threads, if one thread blocks then the entire process blocks and waits for that thread to complete. Since all threads run on one CPU, it can't utilize multiple cores. These are much faster as switches only require library-level access.

Kernel threads are created and managed by the OS. Each thread is known to the OS and scheduled by the OS scheduler. Kernel threads are better positioned to take advantage of multiple cores in comparison to user threads. These threads are slower, as kernel calls are required to switch.

An example where user threads are favored over kernel threads would be a bank ATM system handling many concurrent transactions where low overhead and fast context switching are important.

An example where kernel threads are favored would be for AI applications used to find new drugs where intensive computational work benefits from parallelism across numerous cores of intensive power.
